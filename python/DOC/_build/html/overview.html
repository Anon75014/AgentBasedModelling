Overview of CropWar

Make a Simulation

You can use the :py:func:main.run_full_simulation() function in main.py to simulate an entire episode of the CropWar.

main.run_full_simulation(use_ml_model=False)

Run Simulation of the model and display results

Run a full Simulation of the CropWar model and display resulting graphs and map.txt :param use_ml_model: specify wether a trained machine learning (ml) model should be used, defaults to False :type use_ml_model: bool, optional

Firstly the CropSortiment CropSortiment is initialised and crops are added.

class crops.CropSortiment

Class for crop interaction and tracking

Then the parameters for the CropWar model are specified and the CropwarModel model is initialised. This is based on AgentPy. The agentpy model is then run via the agentpy model-function model.run(). Which performs t_end amount of cycles {}:

setup() -> {update() <-> step()} -> end()

class model.CropwarModel(parameters=None, _run_id=None, **kwargs)

AgentPy model used for simulation.

An agent based model (ABM) to simulate competing farmers. :param parameters: dictionary of parameters, stored as self.p

After that, a graph_class and map_class is initialised to present the result data from the run. The graph class is used to show graphs of interesting parameters recorded during the model update(). The map_class is used to generate a pretty map of the CropWar to visualise the spatial distribution.

Hint

If the parameter save_gif is True in the parameters = {...} then a gif of the temporal map evolution will be saved to \images\map.gif
